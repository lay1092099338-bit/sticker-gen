### 今天修复的 Bug

#### Bug 1 — Excel 嵌入图识别不到参考图
- 根本原因：前端 `generateImages` 没有把 `referenceImageB64` 传给 `/api/generate-variant`；server 也没有接收，直接传 `null` 给模型
- 修复：前端请求加 `referenceImageB64: c.referenceImageB64`；server 接收并传给 `callModelverse`

#### Bug 2 — 拆分文案层没有透明底色
- 原因1：`doSplitLayer` 里 `apiKeyInput` 拼写错误（应为 `apiKey`）
- 原因2：模型无法直接输出透明 PNG
- 修复：改 prompt 让 AI 输出白底纯文字图，前端用 Canvas 做去白底处理（RGB≥230的像素透明度置零）
- 同时修了 `urlToBase64` 里读 `json.b64` 应为 `json.data`

#### Bug 3 — 没有参考图的行识别不出样式
- 根本原因：`extractEmbeddedImages` 用 `getElementsByTagName('xdr:twoCellAnchor')` 带命名空间前缀写法，在 `@xmldom/xmldom` 里有时匹配不到，返回空 map
- 修复：
  1. 改用 localName 匹配（兼容任意前缀）
  2. 同时支持 `oneCellAnchor`（另一种常见锚点类型）
  3. 每行支持两张图：第一张→`__embeddedImage__`（参考图），第二张→`__patternImage__`（图案，自动填 insertedImageB64）
  4. 兜底方案：XML 解析失败时按 `xl/media/` 文件名顺序分配给各行

### Excel 格式约定

### 技术栈
- 后端：Node.js + Express + XLSX + AdmZip + @xmldom/xmldom
- 图像生成：Modelverse API → `gemini-3-pro-image-preview`
- 图像编辑：同上（edit-image 路由）
- 图文拆分：两步 AI 编辑（去文字背景图 + 白底文字图→Canvas去白底）
